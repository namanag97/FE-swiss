Every operations team has dashboards. Average cycle time, throughput, SLA compliance rate. These metrics are useful — but they're symptoms, not causes.

Process mining shows you the *why* behind the numbers.

## The dashboard problem

Here's a typical ops dashboard:

```
  Operations Dashboard
  ==========================================

  Avg Cycle Time     12.4 days    ( -0.3 )
  SLA Compliance     87%          ( -2%  )
  Throughput         1,240/week   ( +5%  )
  Rework Rate        18%          ( +1%  )
```

SLA compliance dropped 2%. Why? The dashboard can't tell you. You open a spreadsheet, start filtering, run some SQL queries, ask three different people what changed. Two days later you might have an answer. Maybe.

## What process mining reveals

The same data, analyzed as a process:

```
  Process Map (Order-to-Cash)
  ==========================================

  +--------+     +----------+     +---------+
  | Order  +---->| Credit   +---->| Fulfill |
  | Entry  |     | Check    |     |         |
  +--------+     +-----+----+     +----+----+
                       |               |
                       v               v
                 +-----+----+    +----+----+
                 | Manual   |    | Ship    |
                 | Review   |    |         |
                 +-----+----+    +----+----+
                       |               |
                       v               v
                 +-----+----+    +----+----+
                 | Approved |    | Invoice |
                 +----------+    +---------+


  Bottleneck: Credit Check -> Manual Review
  Median wait: 4.2 days (was 1.8 days last month)
  Affected cases: 34% of all orders
```

Now you can see it: 34% of orders are hitting the manual review path after credit check, and the wait time there has more than doubled. That single transition is the reason SLA compliance dropped.

## Three things dashboards can't show

### 1. Process variants

The same process runs differently for different cases. Some orders go straight from credit check to fulfillment. Others loop through manual review twice. Dashboards average these together. Process mining separates them.

```
  Variant Analysis
  ==========================================

  Variant 1 (Happy path)          52% of cases
  Entry -> Credit -> Fulfill -> Ship -> Invoice
  Avg duration: 3.1 days

  Variant 2 (Manual review)       34% of cases
  Entry -> Credit -> Review -> Approve -> Fulfill -> Ship
  Avg duration: 8.7 days

  Variant 3 (Rework loop)         14% of cases
  Entry -> Credit -> Review -> Reject -> Review -> ...
  Avg duration: 18.2 days
```

Variant 3 is 6x slower than the happy path. It's also the reason your rework rate is 18%. A dashboard would never surface this.

### 2. Bottleneck location

Average cycle time is 12.4 days. But where is the time being spent?

```
  Time Breakdown by Transition
  ==========================================

  Order Entry -> Credit Check     ##           0.5 days
  Credit Check -> Manual Review   ##########   4.2 days
  Manual Review -> Approved       ####         1.8 days
  Approved -> Fulfill             ##           0.4 days
  Fulfill -> Ship                 #####        2.1 days
  Ship -> Invoice                 ###          1.2 days
```

The bottleneck isn't in fulfillment or shipping — it's in the handoff from credit check to manual review. A dashboard showing average cycle time would never point you to this specific transition.

### 3. Conformance drift

Your intended process says every order should go through credit check. But the discovered model shows 8% of orders skip it entirely — and 3% of those result in payment defaults.

```
  Conformance Report
  ==========================================

  Step              Expected  Actual   Gap
  ----------------  --------  ------   ----
  Order Entry       100%      100%     --
  Credit Check      100%      92%      -8%
  Approval          100%      97%      -3%
  Fulfillment       100%      100%     --
  Shipping          100%      99%      -1%
  Invoicing         100%      100%     --

  8% of cases skip Credit Check
  3% of skipped cases result in payment default
  Estimated annual exposure: $2.4M
```

This isn't a performance problem — it's a compliance risk. And it's invisible in a standard dashboard.

## When to use what

| Question | Tool |
|----------|------|
| What happened last month? | Dashboard |
| How much did we ship? | Dashboard |
| Why did cycle time increase? | Process mining |
| Where are the bottlenecks? | Process mining |
| Are we following the process? | Process mining |
| What will happen to this case? | Process mining + AI |

Dashboards and process mining aren't competitors — they're complements. Dashboards tell you *what*. Process mining tells you *why*.
